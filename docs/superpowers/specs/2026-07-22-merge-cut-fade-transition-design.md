# 合并导出剪切点淡黑过渡设计

## 状态

- 设计日期：2026-07-22
- 目标版本：4.0.7
- 用户已确认：默认开启、固定工具栏位置、总时长默认 `0.46s`、短片段显式拒绝、仅重编码必要边界窗口及安全 GOP 依赖区。

## 目标

为同一源文件的多片段“导出并合并”增加剪切点淡黑过渡：上一片段在末尾线性变暗至黑，下一片段从黑线性恢复亮度。效果不改变片段顺序、选区内容、音频时间线或合并输出总时长，并尽可能保留未受效果影响的源视频包。

## 非目标

- 不做两个片段画面的重叠交叉溶解。
- 不插入额外黑帧，也不缩短片段以腾出转场时间。
- 不做音频淡出、淡入或交叉淡化。
- 不扩大当前精确源保留管线支持的编码、容器或轨道组合。
- 不把用户提供的 `参考/mnrw.mp4` 加入版本控制。

## 参考视频测量

`参考/mnrw.mp4` 为 60 fps、162 帧、2.7 秒的 H.264/AAC MP4，仅包含一个明显淡黑衔接点。

- 上一镜头首次变暗至中央黑场：14 帧，约 `0.233333s`。
- 下一镜头从黑场恢复正常：14 帧，约 `0.233333s`。
- 完整过渡：28 帧，约 `0.466667s`。
- 两侧亮度变化近似线性，帧数完全对称。

按用户确认，界面默认值和持久化默认值采用 `0.46s`，两侧各 `0.23s`。在 60 fps 输出中，滤镜按源帧时间采样，目标视觉长度约为每侧 14 帧。界面不把 `0.46` 自动改写成其他数值。

## 用户界面

新增紧凑组件 `MergeTransitionControl`，固定放在时间线第一行工具栏左侧，准确位置为：撤销、重做和分割按钮之后，波形、缩略图、关键帧等现有按钮之前。对应代码插入点是 `BottomBarFirstRow` 左侧容器中的 `{leadingControls}` 之后。

该组件不依赖当前导出模式显隐：

- 复选开关始终显示，文案为“剪切点淡黑过渡”。
- 默认勾选。
- 勾选时，在开关后显示可编辑数值 `0.46` 和单位 `s`。
- 未勾选时，时长输入和单位一起隐藏。
- 取消勾选不清空时长；再次勾选恢复最近一次合法值。
- 数值输入步进为 `0.01s`，必须是大于 `0` 的有限小数。
- 编辑期间允许暂时为空；按 Enter 或失焦时提交合法值，无效值恢复为最近一次合法值。
- 组件在“单独文件”模式下仍保持可见并可编辑，但设置只在实际多片段合并时生效。

布局必须适应现有工具栏宽度：组件不可挤压中央播放/时间控件；数值输入采用固定紧凑宽度，左侧容器空间不足时由现有整体布局自然收缩，不覆盖相邻按钮。

## 设置与状态

在 `Config` 中增加两个 JSON 可序列化字段：

- `mergeTransitionEnabled: boolean`，默认 `true`。
- `mergeTransitionDuration: number`，默认 `0.46`，单位秒，表示一次剪切点从全亮到黑再恢复全亮的总时间。

设置通过现有 `electron-store`、`useUserSettingsRoot` 和 `UserSettingsContext` 链路持久化。开关关闭时仍保留最后一次合法时长。导出开始时一次性快照开关和时长，导出过程中修改界面只影响下一次导出。

## 时间轴语义

当且仅当以下条件同时满足时应用效果：

1. 实际导出意图是 `merge`；
2. 开关已开启；
3. 至少有两个待合并片段。

设用户输入的总时长为 `T`，单侧时长为 `D = T / 2`。对于每个内部衔接点：

- 上一片段的最后 `D` 秒执行线性淡出至黑。
- 下一片段的最前 `D` 秒执行线性淡入恢复。
- 两侧不重叠，不共享源时间，不插入额外帧。
- 输出总时长严格保持 `sum(segment.end - segment.start)`。
- 音频继续使用原有片段拼接时间线，不应用音量滤镜。

单片段合并没有内部衔接点，因此即使开关开启也不应用滤镜，不改变原导出路径。

## 短片段与输入校验

切点吸附到真实帧 PTS 后、创建 staging 目录或任何输出文件前，使用最终片段时长进行校验：

- 第一个片段至少长 `D`，用于淡出。
- 最后一个片段至少长 `D`，用于淡入。
- 每个中间片段至少长 `T`，用于互不重叠的淡入和淡出。

任何片段不满足要求时，整次导出立即停止，并显示包含片段序号、实际时长和所需最小时长的用户可见错误。不得自动缩短效果、钳制用户值、忽略某个切点或留下部分输出。

## 规划层

新增纯 TypeScript 规划模块 `mergeTransition.ts`，把界面设置和 FFmpeg 命令构造隔离。规划层负责：

- 规范化并验证 `enabled` 和总时长。
- 对每个片段计算 `fadeInDuration`、`fadeOutDuration`。
- 计算效果覆盖的源时间区间。
- 给出现有 source-preserving 计划需要采用的复制目标边界。
- 返回期望总时长和所有内部衔接点在输出时间轴上的位置。

关闭开关、单独文件导出或单片段合并时，规划层返回无效果计划，调用者必须完全沿用现有行为。

## 源包保留与安全 GOP 边界

启用效果的多片段合并强制进入现有精确源保留管线。每个片段的目标复制区按以下方式确定：

- 有淡入时，视频前缀至少编码到淡入结束，并继续到该时间之后最近的安全 H.264 IDR；滤镜只作用于前 `D` 秒。
- 有淡出时，视频后缀从淡出开始之前最近的安全 H.264 IDR 起编码到片段结束；滤镜只作用于最后 `D` 秒。
- 前后编码区重叠时，整段只编码一次。
- 两个编码区之间仍然存在的完整安全 GOP 继续 packet copy。

因此，“效果影响范围”和“为安全拼接必须重编码的依赖范围”是两个概念：亮度只在用户指定的 `D` 秒内变化，编码范围可以为保证独立解码而扩展到安全 IDR。

边界编码继续继承源视频的 codec、bitrate、timebase、B-frame 深度、profile、像素格式、色彩空间、色彩范围、transfer、primaries 和 chroma location。不得用零 B-frame 或不同视频参数换取实现便利。

## FFmpeg 滤镜

在现有 `cutEncodeSmartPart` 的闭合 GOP 编码路径中加入结构化的可选视觉效果参数。滤镜顺序为：

1. 现有 PTS 归零和精确 `trim`；
2. 如需淡入，应用从部件起点开始、时长为 `D` 的线性 black fade-in；
3. 如需淡出，应用在部件末尾前 `D` 秒开始的线性 black fade-out；
4. 保持现有输出时间基和编码参数。

不得使用 `xfade`、额外 color 输入或会改变帧数/总时长的 filter graph。无视觉效果的编码部件不得附加 fade 滤镜。

## 支持边界与错误处理

启用效果的多片段合并继承当前精确导出的硬约束：一个真实视频轨、H.264、MP4/MOV 家族容器、不含外部轨。输入不受支持时明确报错，不自动切换为全量低保真重编码，也不静默导出无效果版本。

staging、验证、原子发布和失败回滚继续使用现有机制。验证或发布失败时，最终目标不得被当作成功结果；进度只有在验证、发布和清理全部完成后才允许显示 `100%`。

## 测试设计

所有生产实现遵循测试先行：

### 纯规划测试

- 默认配置为开启和 `0.46s`。
- 两片段计划得到上一段淡出 `0.23s`、下一段淡入 `0.23s`。
- 三个及以上片段的中间段同时具有互不重叠的淡入和淡出。
- 单片段、关闭开关和 `separate` 意图返回无效果计划。
- 非有限、零或负数时长被拒绝。
- 首段、末段和中间短片段分别在正确边界被拒绝。
- 期望输出总时长始终等于所有片段时长之和。

### 源保留计划与命令测试

- 淡入结束和淡出开始不会落在 packet-copy 区。
- 效果区扩展到安全 IDR 后，滤镜起止仍严格使用 `D`。
- 前后编码区重叠时只生成一个连续编码部件。
- 无效果路径生成的命令与现有路径一致。
- B-frame、timebase、像素格式和色彩参数继续传入边界编码器。

### 真实 FFmpeg 回归

- 生成可重复的 60 fps H.264 测试源，执行两片段合并。
- 验证输出总时长和视频 packet presentation coverage 不变。
- 验证衔接点达到 black，淡出和淡入各约 14 帧，允许一帧采样误差。
- 验证效果区外亮度正常，连接点前后可解码。
- 验证音频时长和原有拼接位置不变。
- 验证完整安全 GOP 仍有源包复用，边界继承源 B-frame 深度。
- 开关关闭后验证无 fade 且现有精确导出结果不回归。

`参考/mnrw.mp4` 只用于本地测量和人工对照。自动化测试使用生成式素材，避免提交用户参考文件。
